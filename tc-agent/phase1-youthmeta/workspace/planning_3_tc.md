## [MBSHIP] 회원가입 / 상태 관리

### MBSHIP-STATUS-001 유스메타 링크 가입 후 상태값 플래그 세팅 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | High |
| 플랫폼 | Web |
| 연관 화면 | 가입 완료 화면 / 관리자 회원 조회 |

**사전 조건**
- 유스메타 링크가 정상 발급된 상태이다
- 신규 가입 계정이 없는 이메일이다
- 관리자 계정으로 회원 DB 조회가 가능한 환경이다

**테스트 단계**
1. 유스메타 링크를 클릭하여 가입 페이지로 진입한다
2. 이메일, 비밀번호 등 필수 정보를 입력하고 가입을 완료한다
3. 관리자 화면 또는 DB에서 해당 회원의 유스메타 가입 상태값 컬럼을 조회한다

**예상 결과**
- 해당 회원의 유스메타 가입 상태값 컬럼이 `true` (또는 활성 상태)로 세팅된다
- 일반 가입 경로로 가입한 다른 회원의 해당 컬럼은 `false` (또는 비활성 상태)이다

3. 관리자 화면 또는 DB에서 해당 회원의 유스메타 가입 상태값 컬럼을 조회한다

**예상 결과**

## [NOTIFY] 알림 기능

### NOTIFY-KAKAO-001 지정 이벤트 발생 시 카카오톡 알림 수신 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | High |
| 플랫폼 | 공통 |
| 연관 화면 | 알림 설정 화면 |

**사전 조건**
- 로그인 상태이며 카카오톡 알림 수신 동의가 완료된 계정이다
- 카카오톡 연동이 설정되어 있다
- 알림 발송 트리거로 지정된 이벤트가 정의되어 있다

**테스트 단계**
1. 알림 트리거로 지정된 이벤트를 발생시킨다 (예: 포지션 청산)
2. 해당 계정과 연결된 카카오톡을 확인한다

**예상 결과**
- 이벤트 발생 후 수 초 이내에 카카오톡 알림 메시지가 수신된다
- 알림 메시지에 이벤트 내용이 정확히 포함되어 있다

**비고**
- 알림 발송 트리거 목록 확정 후 각 트리거별 개별 테스트 필요 (미결 사항)

- 주문 체결 후 포지션에 TP/SL이 설정되지 않는다 (빈 상태)

### TPSL-PART-001 추가 주문마다 Partial TP/SL 누적 생성 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | Medium |
| 플랫폼 | Web |
| 연관 화면 | 포지션 / TP/SL 목록 화면 |

**사전 조건**
- 로그인 상태이다
- 자동 TP/SL 설정이 ON 상태이다
- 해당 코인에 이미 초기 주문이 1개 체결된 포지션이 있다

**테스트 단계**
1. 동일 코인에 추가 주문을 실행한다
2. 포지션 또는 TP/SL 목록 화면에서 TP/SL 개수를 확인한다
3. 추가 주문을 한 번 더 실행한다
4. TP/SL 목록 화면에서 TP/SL 개수를 재확인한다

**예상 결과**
- 추가 주문마다 Partial TP/SL이 1개씩 생성되어 누적된다

**비고**
- Partial TP/SL 최대 개수 또는 병합 정책 확정 후 상한선 케이스 테스트 추가 필요 (미결 사항)

### TPSL-SET-003 파트너 계정과 개인 계정의 TP/SL 기본값 차이 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | High |
| 플랫폼 | Web |
| 연관 화면 | 사용자 설정 화면 / 포지션 화면 |

**사전 조건**
- 파트너 계정과 개인 계정이 각각 존재한다
- 두 계정 모두 자동 TP/SL이 ON 상태이다
- TP/SL 기본값이 파트너/개인별로 다르게 설정되어 있다 (기본 ±10%)

**테스트 단계**
1. 파트너 계정으로 로그인하여 주문을 실행하고 TP/SL 값을 확인한다
2. 개인 계정으로 로그인하여 동일 조건의 주문을 실행하고 TP/SL 값을 확인한다

**예상 결과**
- 파트너 계정과 개인 계정의 자동 TP/SL 기본값이 설정에 따라 다르게 적용된다

**비고**
- 정확한 수치는 기획 확정 후 검증 값 업데이트 필요 (미결 사항)
