## 변경된 주요 규칙 (v3→v4 차이 원인)

### 1. 커버리지 임계값 상향 (60% → 95%)
- v3: tc-reviewer가 60% 커버리지에서 통과 → 21개 기능 미커버
- v4: 95% 임계값 적용 → 모든 86개 기능이 1개 이상 TC로 커버됨

### 2. 도메인 코드 체계 재정립
- v3: `AUTH-ONBRD-*`, `AUTH-IOSBR-*`, `AUTH-ANBR-*` 등 비표준 코드 사용
- v4: `AUTH-ONBD-*`, `AUTH-INAPP-*` 등 규칙 기반 표준 코드 적용
- v3: `FUND` 도메인 독립 → v4: `AUTH-FUND-*`로 통합

### 3. TC 제목 규칙 강화
- v3: 자유 형식 제목 (예: "레버리지 제한 동작 확인")
- v4: 반드시 `~확인`으로 종결 (예: "레버리지 슬라이더 최대값 2배 제한 확인")

### 4. THEN 절 품질 기준 강화
- v3: "정상적으로 처리된다", "제대로 표시된다" 등 모호한 표현 허용
- v4: 측정 불가 표현 금지, 구체적 화면 관찰 기준 명시

### 5. [미결] 태그 의무화
- v3: 미결 사항이 비고에 자유 텍스트로 산발적 기재
- v4: `[미결]` 태그 형식 표준화, 35건 체계적 관리

### 6. [신규] 태그 도입
- v3: [신규] 개념 없음 (기획서 기반 TC만)
- v4: GitHub 코드 분석으로 도출된 TC에 `[신규]` 태그 24건 부착

### 7. ★ 최소 TC 세트 선별 기준 도입
- v3: 최소 TC 세트 미정의
- v4: High 우선순위 + Happy Path + 환경 분기 기준으로 70개 선별

### 8. 4-배치 도메인 분할 작성
- v3: 단일 tc-writer로 전체 작성 → rate limit으로 55개에서 중단, 이후 76.5% 커버리지로 통과
- v4: 도메인별 4개 배치 병렬 작성 → rate limit 회피, 완전 커버리지 달성
