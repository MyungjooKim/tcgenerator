## 기술 고려사항

- **동시성**: 레버리지 자동 동기화와 사용자 수동 변경이 동시에 발생할 경우 충돌 방지 로직 필요
- **멀티 탭/브라우저**: WebSocket 또는 Polling 방식의 상태 공유 전략 결정 필요
- **거래소별 분기**: Bybit, Binance, Gate 등 거래소마다 레버리지 API 스펙이 달라 추상화 레이어 필요
- **Partial TP/SL**: 거래소별 Partial TP/SL 지원 여부 사전 확인 필요
- **사용자 연령대**: 50대 이상 대상 → UI 텍스트 명확성, 오류 안내 문구 쉬운 표현 필수
- **Available Balance**: 레버리지 변경 시 AB 재계산 로직 거래소 API 기반 확인 필요
