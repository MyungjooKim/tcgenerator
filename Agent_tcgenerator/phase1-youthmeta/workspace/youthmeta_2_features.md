## 기술 고려사항

- **파트너 코드 분기**: `partner_code = YOUTHMETA` 여부에 따라 레버리지 UI, 모달 동작이 달라지므로 계정 레벨 feature flag 관리 필요
- **거래소별 TP/SL API**: OKX, GATE 거래소의 Partial TP/SL 지원 여부 사전 확인 필요
- **상태 동기화**: Settings 토글과 Order Form 토글이 실시간 동기화되어야 함 (전역 상태 관리)
- **지정가 주문 TP/SL 생명주기**: 주문 생성 → 미체결 대기 → 체결 → 포지션 TP/SL 전환 과정에서 상태 추적 필요
- **Auto 배지 상태 관리**: 자동 생성 여부 플래그를 주문/포지션 데이터에 포함하여 관리
- **로컬 저장소**: 최초 로그인 팝업 확인 이력은 로컬스토리지에 저장 (계정별 키로 관리)
- **Binance 추가(5월)**: 거래소별 TP/SL API 스펙 상이, 추상화 레이어 필요
