## 기술 고려사항

- **포인트 집계 주기**: Overview의 Total Points는 "updated daily"로 명시. History의 Active 주차 데이터는 일정 주기 동기화 (정확한 주기 미정)
- **거래량 집계**: 기본 포인트는 1시간마다 집계 (종합 시나리오 기준)
- **레퍼럴 커미션**: 실시간 지급 (Real Time)
- **X API 연동**: 포지션 공유 미션의 X(Twitter) 포스팅 완료 감지 필요
- **다양성 추구 미션 수치 불일치**: 기획서 개요(5개)와 트리거/처리(3개) 간 불일치 → 최종 수치 확정 필요
- **레퍼럴 등급 업데이트 주기**: "Trading volume updates are not real-time" 안내 문구 존재 → 집계 주기 확정 필요
- **Season 전환**: 레퍼럴 등급 초기화 시 커미션율 즉시 재적용 필요
