## [포인트 적립 — 기본 거래 포인트]

### 정책 목표
- 거래, 유지, 확산 세 축을 중심으로, 단순 거래 보상을 넘어 사용자가 플랫폼에 머물고 스스로 성장하는 재미를 느끼도록 설계된 포인트 프로그램

### 핵심 정책 규칙

**기본 거래 포인트**
- $2,000 거래량당 1 포인트 적립
- 공식: Trading Volume ÷ 2000 = Base Point
- 소수점 2자리까지 지원, 3자리에서 반올림
- 예: $8,000 → 4.00P, $10,500 → 5.25P

**부스트 시스템 (합연산)**
- 모든 부스트는 합연산으로 적용 (예: +10% + +5% + +10% = +25% → 1.25x)
- 최종 포인트 = Base Point × (1 + 합산 부스트율)

**부스트 종류**
| 부스트 명칭 | 조건 | 부스트율 | 비고 |
|---|---|---|---|
| Referral Boost | 레퍼럴 코드로 가입 시 영구 적용 | +10% | 코드 등록 익일부터 적용 |
| Weekly Task Boost | 주간 미션 전체 완료 시 | +5% | 다음 주부터 자동 적용 |
| Loyalty Boost | 주간 미션 연속 달성 (최대 4주) | 최대 +15% | 2주: +5%, 3주: +10%, 4주: +15%. 한 주 실패 시 0% 초기화 |
| 사전 등록 Boost | 사전 등록 유저, 서비스 개시 후 3주 한정 | +100% | 기간 만료 시 자동 소멸 |

## [데일리 미션]

### 정책 목표
- 매일 00:00 (UTC) 초기화. 가벼운 행동 유도로 DAU 유지

### 핵심 정책 규칙

| 미션명 | Trigger | 완료 처리 | 기획 의도 |
|---|---|---|---|
| 출석 체크 | 지갑 연결 후 메인 대시보드 로드 | 24시간 내 첫 로그인 시 즉시 완료 | 기본 트래픽 확보 |
| 시장 정찰 | Market(시장가) 주문 체결 발생 | 주문 체결 시점에 완료 (생성 아님) | 최소 거래 유도 |
| 가벼운 손풀기 | 당일 누적 거래대금 $100 이상 | 레버리지 포함 총 포지션 규모 기준 ($20×5배 = $100으로 처리) | 최소 거래 유도 |
| 포지션 공유 | PnL 카드 'Share' 클릭 후 X(Twitter) 리다이렉트 | X API 연동, 포스팅 시점에 완료 | 바이럴 마케팅 |
| 승부사 | 레버리지 5x 이상 주문 체결 | 지정가/시장가 무관, 체결 시점에 완료 | 수수료 높은 거래 유도 |

- **모든 미션 완료 시**: 자동으로 1P 즉시 지급
- **초기화**: 매일 00:00 UTC 전체 리셋

## [주간 미션]

### 정책 목표
- 매주 월요일 00:00 (UTC) 초기화. 헤비/라이트 유저를 구분하는 심화 과제

### 핵심 정책 규칙

| 미션명 | 조건 | 완료 처리 | 기획 의도 |
|---|---|---|---|
| 개근상 | 일주일(월~일) 중 '웹 접속' 미션 완료 일수 5일 이상 | 5회 카운트 시 완료 (3/5 형태 노출) | 리텐션 강화 |
| 볼륨 마스터 | 주간 누적 거래대금 $50,000 이상 | $50,000 달성 시 완료 ($20,000/$50,000 형태 노출) | 거래량 증대 |
| 다양성 추구 | 서로 다른 심볼(Ticker) 거래 체결 3건 이상 | BTC, ETH, SOL 각 1회 이상 시 완료 (2/3 형태 노출) | 다양한 마켓 활성화 |
| 멀티 거래소 | 서로 다른 CEX/DEX API 통해 거래 2건 이상 | 2개 이상 거래소 거래 확인 시 완료 (2/2 형태 노출) | 애그리게이터 특장점 |
| 헌터 | 주간 Realized PnL 합계 $100 이상 (손실 차감) | 순수 실현 수익 $100 달성 시 완료 (99/100 형태 노출) | 실력 기반 보상 |

- **모든 미션 완료 시**: 자동으로 5P 즉시 지급
- **초기화**: 매주 월요일 00:00 UTC 전체 리셋

## [레퍼럴 시스템]

### 정책 목표
- 성장형 레퍼럴: 내가 초대한 사람들의 거래량이 늘어날수록 내 등급 상승

### 핵심 정책 규칙

**레퍼럴 등급 (Tier) 및 커미션**
| 등급 | 피추천인 합산 거래량 조건 | 커미션 (친구 획득 포인트 대비) |
|---|---|---|
| Bronze | $0 ~ $1M | 5% |
| Silver | $1M ~ $10M | 10% |
| Gold | $10M ~ $20M | 15% |
| Diamond | $20M~ | 20% |

- 초대자(Referrer): 피추천인 거래 실적에 따라 포인트 커미션 수령
- 피초대자(Referee): 가입 즉시 Referral Boost +10% 적용

**레퍼럴 코드**
- 계정 생성 즉시 자동 생성 (영문 대문자+숫자, 7자, 유일성 보장)
- 커스텀 코드 변경 규칙: 영문+숫자, 최소 4자~최대 10자, 1회만 변경 가능
- 파트너 코드로 등록된 코드는 일반 유저가 커스텀 코드로 사용 불가

**레퍼럴 링크 처리**
- 신규 유저: 레퍼럴 링크 클릭 후 가입 시 자동 등록
- 기가입 유저+미등록: 레퍼럴 링크 클릭 후 로그인 시 자동 등록
- 기가입 유저+이미 등록: 코드 변경 불가 (별도 표시 없음)
- 자신의 레퍼럴 링크: [REF] 표시

**레퍼럴 코드 등록 모달 유효성 검증**
- 유효하지 않은 코드: "invalid referral code" 에러 표시
- 이미 등록 완료 상태: 등록된 코드 표시, Apply 버튼 비활성화

## [파트너 시스템]

### 핵심 정책 규칙
- 파트너: 별도 계약으로 수수료 분배 구조. 파트너 등록 유저는 페이백 기능 이용 불가
- 포인트 프로그램은 파트너 유저도 동일하게 이용 가능
- 파트너 코드는 회사명/단체명/활동명 기준으로 설정 (운영 관리)
- 파트너 코드와 동일한 코드는 일반 유저가 커스텀 레퍼럴 코드로 사용 불가

## [Season 제도]

### 핵심 정책 규칙
- 시즌 1: 서비스 개시일부터 약 2개월 (외부 비공개, 상황에 따라 유연 조정)
- 시즌 2 전환 시 초기화 항목: 레퍼럴 등급(티어) (예: Gold → Bronze 초기화)
- 포인트 자체는 초기화되지 않음 (기획서 상 명시 없음)

**사전 등록 부스트 기한**
- 서비스 개시일부터 3주 동안 적용
- 개시 전 사전 등록 이메일에 안내 메일 발송 필요
- 3주 경과 시 자동 소멸

## [포인트 페이지 UI]

### 핵심 정책 규칙

**Overview 영역**
- 상단 요약 카드: My Total Points, Boost(합산%), Referral link(복사 아이콘)
- "Your points are updated daily" 안내 문구
- [Learn more]: 깃북 문서 새 탭 출력
- 미션 영역: Daily Mission / Weekly Mission 토글 (남은 시간 HH:MM 또는 D:HH 실시간 표시)
- Boosts 영역: 4가지 부스트 카드 (활성/비활성 상태)
- Referral / History 탭 영역

**미션 완료 오버레이**
- 모든 체크박스 활성화 즉시 오버레이 등장 + 포인트 즉시 지급
- 오버레이에 리셋 카운트다운 HH:MM:SS 표시
- 초기화 시 오버레이 사라지고 모든 미션 빈 체크박스로 갱신

**Boosts 영역 세부**
- 사전 등록 Boost: 활성 시 'Boost Active' + 남은 기간 표시
- Referral Boost: 미등록 유저에게 "Go to Register Referral Code" CTA 노출. 활성 시 'The boost will be applied starting the next day' 안내
- Weekly Task Boost: 활성 시 'Boost Active' + 남은 기간 + "View Weekly Missions" 링크
- Loyalty Boost: 주차별 성공/실패 타임라인 (week 1~6 형태), 결과 없는 주차는 별도 아이콘, 현재 적용 % 표시

**레퍼럴 탭 세부**
- Referral 탭 (기본) / History 탭 선택 가능
- 레퍼럴 등급(Tier): 현재 등급 표시
- 진척도(Progress to Next Tier): 바 형태, 현재 거래량 / 다음 티어 기준 표시
- 레퍼럴 코드: 자동 생성 코드 표시
- Custom Code 버튼: 클릭 시 커스텀 코드 변경 모달 출력
- 레퍼럴 링크: URL 표시 + 복사 아이콘
- Enter Referral Code 버튼: 클릭 시 코드 등록 모달 출력
- Referral Performance 테이블: Rank, Address (마스킹), Referred On, Trading Volume In Supercycl (상위 거래량 순 정렬, 10개/페이지)
- 페이지네이션: 10개/페이지, 10개 미만 시 해당 행 미노출

**레퍼럴 코드 등록 모달**
- 코드 입력 후 Apply → 유효성 검증 → 성공 시 등록 및 모달 닫힘
- 유효하지 않은 코드: "invalid referral code" 에러
- 이미 등록된 경우: 등록된 코드 표시, 버튼 비활성화

**커스텀 코드 변경 모달**
- 영문+숫자, 4~10자 입력, Save 클릭 시 유효성 검증
- 중복 코드: "Already used" 에러
- 이미 1회 변경 이력 있을 경우: 버튼 비활성화 (변경 불가)
- 글자 수 카운터 표시 (예: 10/20)

**History 탭 세부**
- 열: Week(번호), Status(Active/Finished), Period(기간), Boost(%), Total Points(소수점 2자리)
- 정렬: 최신 주차(Active) 최상단, 내림차순
- 기간 기준: 매주 월요일 09:00 ~ 다음 주 월요일 08:59
- 페이지네이션: 10개/페이지
- Active 주차 데이터는 일정 주기로 동기화

**FAQ 탭**
- Q: How do I earn points? A: 거래 시 자동 지급
- Q: How often are points distributed? A: 매일 00:00 UTC
- Q: What are points used for? A: 토큰 에어드랍 배분

### 미결 사항
- 포지션 공유 미션의 X API 실패 시 처리 방안
- 다양성 추구 미션: 기획서 내 "5개 이상" vs "3개 이상" 불일치 → 정확한 수치 확정 필요
- History 탭의 포인트 정산 집계 주기 확정
- 시즌 2 전환 시 포인트 이월 여부 미명시
- 커스텀 코드 변경 후 기존 코드로 유입된 레퍼럴 처리 방안
