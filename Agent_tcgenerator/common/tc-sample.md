# TC 샘플 — Supercycl 모바일 체험

> 이 파일은 tc-writer가 스타일·상세도·분해 수준을 맞추기 위한 기준 예시입니다.
> 각 도메인에서 대표 케이스를 선별했습니다. 이 스타일과 상세도를 유지하세요.

---

## 샘플 선택 기준

- **Positive**: 단계가 명확하고, THEN이 화면에서 직접 확인 가능한 수준
- **Negative**: 구체적인 오류 메시지 또는 비활성화 상태를 명시
- **Edge**: [미결] 태그 사용 방법 포함
- **분리 vs 합치기**: TC-001~003은 동일 기능을 올바르게 3개로 분리한 예시

## [AUTH] 인증/온보딩

### **AUTH-TERMS-001** — 약관 동의 후 온보딩 진입 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | High |
| 플랫폼 | Web(Mobile) |
| 연관 화면 | /terms |

**사전 조건**
1. Google OAuth 로그인 완료된 상태
2. 약관 미동의 상태
3. `/terms` 페이지에 접속된 상태

**테스트 단계**
1. 약관 내용이 화면에 표시되는지 확인한다
2. 약관 동의 체크박스를 탭한다
3. `Accept` 버튼을 탭한다

**예상 결과**
- 약관 동의 상태가 서버에 저장된다
- 온보딩 페이지(`/onboarding`)로 이동한다

### **AUTH-TERMS-002** — 체크박스 미선택 시 Accept 버튼 비활성화 확인

| 항목 | 내용 |
|------|------|
| 분류 | Negative |
| 우선순위 | High |

**사전 조건**

- 버튼을 탭해도 화면 이동이 발생하지 않는다

### **AUTH-ONBRD-001** — 온보딩 3단계 자동 진행 성공 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | High |
| 플랫폼 | Web(Mobile) |
| 연관 화면 | /onboarding |

**사전 조건**
1. Google OAuth 로그인 완료된 상태
2. 약관 동의가 완료된 상태
3. `/onboarding` 페이지에 접속된 상태

**테스트 단계**
1. 온보딩 화면이 표시되는지 확인한다
2. 지갑 생성 진행 상태(단계 1)가 표시되는지 확인한다

4. 자금 로딩 진행 상태(단계 3)가 표시되는지 확인한다
5. 온보딩 완료 화면이 표시될 때까지 대기한다

**예상 결과**
- 3단계(지갑 생성 → Testnet 연결 → 자금 로딩)가 순차적으로 진행 상태와 함께 표시된다
- 완료 화면에 초기 잔액이 표시된다
- `Start Trading` 버튼이 표시된다

**비고**
- [미결] 초기 지급 잔고 금액: PDF 기준 1,000 USDC / 코드 기준 10,000 USDC — 정책 확정 필요

## [LEVR] 레버리지 — TC 분리 예시 (올바른 분해 방법)

> 아래 3개 TC는 "LeverageNotice 팝업" 기능을 올바르게 3개로 분리한 예시입니다.
> 팝업 표시 / 버튼 동작 / 재방문 상태는 사전 조건이 다르므로 분리합니다.

### **LEVR-POPUP-001** — LeverageNotice 팝업 최초 노출 확인

| 항목 | 내용 |
|------|------|
| 분류 | Positive |
| 우선순위 | High |
| 플랫폼 | Web(Mobile) |
| 연관 화면 | /trade |

**사전 조건**
1. YOUTHMETA 파트너코드로 가입한 계정인 상태
2. 온보딩 완료된 상태
3. TradingPage 최초 진입 전인 상태 (LeverageNotice 미확인)

**테스트 단계**
1. 온보딩 완료 후 `Start Trading`을 탭하여 TradingPage에 진입한다
2. 화면에 팝업이 자동 표시되는지 확인한다
3. 팝업 내 문구와 버튼을 확인한다

**예상 결과**
- LeverageNotice 팝업이 자동으로 표시된다
- 팝업에 "I Understand" 버튼이 표시된다

**비고**
- [미결] 팝업 내 표시 언어(한국어/영어) 미확정 — 정책 확정 후 문구 검증 필요

## 스타일 가이드 요약

| 항목 | 기준 |
|------|------|
| **사전 조건 형식** | 번호 붙은 개조식. 설정 순서대로. "~인/된/한 상태"로 끝맺음. "이다" 금지 |
| **사전 조건 순서** | 계정 상태 → 화면/탭 진입 → UI 조작 완료 순 |
| 테스트 단계 수 | 보통 2~5단계. 6단계 초과 시 TC 분리 고려 |
| 예상 결과 수 | 보통 1~3개. 4개 초과 시 TC 분리 고려 |
| THEN 표현 깊이 | 화면에서 눈으로 확인 가능한 것만 |
| 코드 내부 변수 | THEN에 쓰지 않고, [신규] 비고란에만 참고로 표기 |
| 오류 메시지 | 기획서/코드에 명시된 텍스트는 그대로 사용 |
| [미결] 태그 | 정책 미확정 시 반드시 비고에 명시 |
| [신규] 태그 | 코드에서만 발견된 TC에 부여 |
| 추가 주석 | Edge 조건 설명은 해당 항목 뒤에 `(* ...)` 형태로 붙임 |
